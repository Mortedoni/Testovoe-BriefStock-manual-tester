
~~~~~~~~~~ Задача ~~~~~~~~~~

Проверить результат выполнения ТЗ в файле chat.html. Сделать отчеты о проведенном тестировании (тест кейзы, чек-листы, баг репорты).

В рамках тестового задания сообщение можно считать успешно отправленным, если после ввода оно появилось в списке сообщений.

Результат оформить в удобном формате.

~~~~~~~~~~  Техническое задание (ТЗ) для web чата: ~~~~~~~~~~~~~

1) Реализовать web чат.

2) В чате должна быть реализована отправка сообщений и отправка файлов.

3) После отправки файла в чате должно появиться сообщение с информацией об отправленном файле: имя и размер файла.

3) Если в поле набора сообщения есть текст, кнопка отправки сообщения должна быть активной. Если нет - неактивной, disabled.

4) После набора сообщения, при нажатии на Enter должен происходить только перенос на новую строку.

5) После набора сообщения, при нажатии на Ctrl + Enter должна происходить отправка сообщения.

6) Окно чата должно быть фиксированных размеров, при большом количестве сообщений должен появляться вертикальный скролл.

7) Над чатом должно быть расположено поле для поиска сообщений.

8) При вводе каждого символа в поле поиска сообщений, сообщения в чате должны отфильтровываться в соответствии с введенным поисковым запросом.

9) Страница логина должна быть расположена по пути /chat.html?login

На этой странице пользователь должен ввести свое имя, после чего происходит 

перенаправление на страницу /chat.html?chat

10) Страница с чатом должна быть расположена по пути /chat.html?chat

11) Под чатом должна распологаться ссылка, при клике на которую начинается загрузка файла с историей чата

12) Окно чата должно быть адаптивным - подстраиваться под ширину окна браузера, но не шире 600px.

Не должно появляться горизонтального скролла если ширина окна браузера больше 200px.

13) На странице /chat.html?chat, самым верхним элементом должно быть лого нашего сайта, взятое из файла logo.webp

Лого должно быть полностью видно на странице, без потери качества.

14) Длина одного сообщения в чате не должна превышать 60 символов с учетом пробелов

Если превышает - то сообщение разбивается на несколько сообщений и вместо одного одновременно отправляется несколько

15) Не должно быть возможности отправлять пустые сообщения без содержимого ( без текста или файла )

16) Пробелы в конце и начале сообщения должны обрезаться у отправленного сообщения

17) На странице /chat.html?chat, под лого должно располагаться приветственное сообщение пользователю в формате "Hello, %ИМЯ_ПОЛЬЗОВАТЕЛЯ% !"
